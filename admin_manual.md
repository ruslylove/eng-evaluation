### โครงสร้าง Google Sheets และการจัดระเบียบโฟลเดอร์ Google Drive

## 2. ฐานข้อมูลบุคลากร (Staff Database)

**Spreadsheet ID:** `1Ljo361CBa1p9d79V4dxOGkttubCo2TnN2eiHwSAKFkk`

สเปรดชีตนี้เป็น **ต้นทางของข้อมูลบุคลากร** ที่ระบบจะดึงมาแสดงในขั้นตอนที่ 1 ทันทีที่ผู้ใช้ล็อกอินด้วยอีเมลมหาวิทยาลัย (จับคู่ด้วยคอลัมน์ Email) ผู้ดูแลระบบต้องอัปเดตชีตนี้เมื่อมีบุคลากรใหม่ / เปลี่ยนตำแหน่ง / เปลี่ยนหัวหน้าภาควิชา

### แท็บ `Staff`
แถวแรกเป็น header — สคริปต์อ่านคอลัมน์ทั้งหมดแบบ dynamic (`staff[header] = value`) ดังนั้น **สามารถมีคอลัมน์เพิ่มเติมได้อย่างอิสระ** แต่คอลัมน์ต่อไปนี้ถูกใช้งานโดยตรงในโค้ด:

| คอลัมน์ | ใช้ทำอะไร |
|---|---|
| `Email` | คีย์จับคู่กับอีเมล Google SSO ของผู้ใช้ (ต้องตรงกับอีเมล `@eng.kmutnb.ac.th` / `@kmutnb.ac.th`) |
| `Div_ID` | เชื่อมไปยังแท็บ `Division` เพื่อดึงชื่อภาควิชา |
| `Title` / `คำนำหน้า` | คำนำหน้าชื่อ (นาย/นาง/นางสาว/ดร.) — ใช้ประกอบชื่อเต็มทางวิชาการ |
| `Academic_Title` / `คำนำหน้าทางวิชาการ` | ใช้ตรวจว่ามีคำว่า "ดร." หรือไม่ |
| `Fullname` / `Fullname_TH` / `ชื่อ` / `Name` | ชื่อจริง |
| `Surname` / `Surname_TH` / `นามสกุล` | นามสกุล |
| `Position` / `ตำแหน่ง` / `Academic_Position` | ตำแหน่งทางวิชาการ (ต้องเป็นค่าตรงตัวใดค่าหนึ่งใน `อาจารย์`, `ผู้ช่วยศาสตราจารย์`, `รองศาสตราจารย์`, `ศาสตราจารย์`, `ศาสตราจารย์ต่อเวลา` เพื่อให้ `CRITERIA` ในโค้ดจับคู่เกณฑ์ได้ถูกต้อง) |

> ⚠️ **สำคัญ:** การค้นหาคอลัมน์ทำแบบ "fuzzy match" ต่อชื่อ header (ดูฟังก์ชัน `findColIndex` ใน `Code.gs`) โดยเทียบทั้งภาษาไทย/อังกฤษและคำที่ประกอบกันบางส่วน หากตั้งชื่อ header ใหม่ ให้เลี่ยงคำที่ทับซ้อนกับคำค้นหาที่มีอยู่ (เช่นอย่าตั้งคอลัมน์ที่มีคำว่า "position" ปนอยู่ถ้าไม่ใช่ตำแหน่งทางวิชาการจริง) และคอลัมน์ `Email` ต้องมีค่าเสมอ ไม่เช่นนั้นผู้ใช้จะล็อกอินเข้าระบบไม่ได้ (จะได้ค่า `null` จาก `getStaffProfile()`)

### แท็บ `Division`
ใช้เชื่อมจาก `Div_ID` ของแท็บ `Staff` เพื่อดึงชื่อภาควิชาและอีเมลหัวหน้าภาควิชาสำหรับลงนามในบันทึกข้อความ

| คอลัมน์ | ใช้ทำอะไร |
|---|---|
| `Div_ID` | คีย์จับคู่กับ `Staff.Div_ID` |
| `Name` | ชื่อภาควิชาภาษาไทย (แสดงในโปรไฟล์ผู้ใช้ เป็น `Div_Name_Th`) |
| `Short_Name` | ชื่อย่อภาควิชา (`Div_Short_Name`) |
| `Eng_Name` | ชื่อภาควิชาภาษาอังกฤษ (`Div_Name_En`) |
| `Head_of_Department` / `Head_Email` / `หัวหน้าภาควิชา` | **อีเมล** ของหัวหน้าภาควิชา — ใช้ค้นชื่อเต็มหัวหน้าภาควิชากลับไปที่แท็บ `Staff` เพื่อเติมชื่อผู้ลงนามในบันทึกข้อความอัตโนมัติ (`{{DEPT_HEAD}}`) |

> เมื่อเปลี่ยนหัวหน้าภาควิชา ให้แก้ **อีเมล** ในคอลัมน์นี้เท่านั้น ระบบจะไปดึงชื่อ-ตำแหน่งล่าสุดจากแท็บ `Staff` โดยอัตโนมัติ ไม่ต้องพิมพ์ชื่อซ้ำสองที่

## 3. ฐานข้อมูลคำขอ (Submissions Database)

**Spreadsheet ID:** `1Nyp_b-tPRxSWSE7H_WaiiFrIxjxqqQmrA0ooV1kmaZ8`

## 4. โฟลเดอร์ไฟล์แนบใน Google Drive

**Root Folder ID:** `1eFqtjXxYhLJyDi_ZZ0R-_B_K1_k7XEMS`

ทุกครั้งที่มีการยื่นคำขอพร้อมไฟล์แนบ สคริปต์จะสร้างโครงสร้างโฟลเดอร์ย่อยให้อัตโนมัติผ่าน `getOrCreateSubfolder_()` (สร้างใหม่เฉพาะเมื่อยังไม่มีโฟลเดอร์ชื่อนั้นอยู่ — จะไม่สร้างซ้ำ):

```
📁 Attachments Root Folder (1eFqtjXxYhLJyDi_ZZ0R-_B_K1_k7XEMS)
 └── 📁 ปีงบประมาณ {Fiscal_Year}         เช่น "ปีงบประมาณ 2568"
      └── 📁 รอบที่ {Round}               เช่น "รอบที่ 1"
           └── 📁 {ชื่อ-นามสกุลไทยของผู้ยื่น}   (fallback: ส่วนหน้า @ ของอีเมล ถ้าไม่มีชื่อ)
                └── 📄 ไฟล์แนบแต่ละรายการ
```

**รูปแบบชื่อไฟล์:**
```
{ส่วนหน้าอีเมล}_FY{ปีงบประมาณ}_R{รอบ}_pub{ลำดับผลงาน}_{ประเภทเอกสารแนบ}_{ชื่อไฟล์ต้นฉบับ}
```
เช่น `somchai.j_FY2568_R1_pub2_proof_of_ranking_scopus_q1.pdf`

**สิทธิ์การแชร์ไฟล์:** ทุกไฟล์ที่อัปโหลดจะถูกตั้งค่าเป็น `ANYONE_WITH_LINK` / `VIEW` โดยอัตโนมัติ (`setSharing`) เพื่อให้ลิงก์ที่ฝังในสเปรดชีต (คอลัมน์ `Attachments`) เปิดดูได้จากทุกที่โดยไม่ต้องขอสิทธิ์เพิ่ม — **นี่หมายความว่าใครก็ตามที่มีลิงก์สามารถเปิดไฟล์ได้ แม้ไม่ได้ล็อกอิน** หากมีความอ่อนไหวของข้อมูล (เช่น เอกสารมีข้อมูลส่วนบุคคล) ควรพิจารณาเปลี่ยนเป็นแชร์เฉพาะภายในโดเมนมหาวิทยาลัยแทน

หากการยื่นล้มเหลวระหว่างสร้างโฟลเดอร์ (เช่นไม่มีสิทธิ์เข้าถึง Root Folder) สคริปต์จะ fallback ไปเก็บไฟล์ไว้ที่ root folder ตรงๆ แทนที่จะแยกโฟลเดอร์ย่อย (`catch` ใน `submitApplication()`) — หากพบไฟล์กระจัดกระจายอยู่ที่ root โดยไม่มีโฟลเดอร์ปี/รอบ ให้ตรวจสอบสิทธิ์ของบัญชีที่รันสคริปต์ก่อน

## 5. เทมเพลตเอกสารบันทึกข้อความ (Docx Template)

**Google Doc ID:** `1RDRb9Wgxvr9M9_D9Eb9mSxewq3w3c7N1wpSGz4Ujj3Q`

เอกสารต้นแบบที่ `generateDocxMemo()` จะ **คัดลอก** (ไม่แก้ไขต้นฉบับ) แล้วแทนที่ placeholder ด้วยข้อมูลจริง จากนั้น export เป็น `.docx` และลบไฟล์สำเนาชั่วคราวทิ้ง (`docFile.setTrashed(true)`)

### Placeholder ที่สคริปต์แทนที่อัตโนมัติ

| Placeholder | แทนที่ด้วย |
|---|---|
| `( {{FULL_NAME}} )` (ในวงเล็บ) | ชื่อเต็มทางวิชาการของผู้ลงนาม (บล็อกลายเซ็น) |
| `{{SIGN_NAME}}` | เหมือนด้านบน |
| `{{FULL_NAME}}` | ชื่อ-นามสกุลไทยของผู้ยื่น (เนื้อหาทั่วไปในเอกสาร) |
| `{{POSITION}}` | ตำแหน่งทางวิชาการ |
| `{{DEPT}}` | ภาควิชา |
| `{{STAFF_TYPE}}` | ประเภทบุคลากร |
| `{{APPOINT_DATE}}` | วันที่แต่งตั้งตำแหน่ง |
| `{{ROUND}}` | เฉพาะในย่อหน้า/เซลล์ตารางที่มีคำว่า "เรื่อง" + "ขอเสนอผลงาน" จะถูกแทนด้วย "รอบที่ N ของ {ชื่อเต็มทางวิชาการ}" — ที่อื่นแทนด้วยเลขรอบเฉยๆ |
| `{{FISCAL_YEAR}}` | ปีงบประมาณ |
| `{{DEPT_HEAD}}` | ชื่อเต็มหัวหน้าภาควิชา (ดึงจาก Staff Database ผ่าน `getDeptHeadName()`) |
| `{{DATE}}` | วันที่ปัจจุบันแปลงเป็นรูปแบบไทย (เช่น "7 กรกฎาคม 2569") |

### ตารางผลงานในเอกสาร
สคริปต์จะค้นหาตารางที่มี header เป็นคำว่า "ลำดับ", "ผลงาน", หรือ "ชื่อ" (fallback: ตารางสุดท้ายในเอกสาร) แล้ว:
1. ลบคอลัมน์ "ผู้แต่ง/ผู้เขียน" ออกถ้ามี
2. เพิ่มคอลัมน์ "คะแนน (%)" ต่อท้าย header ถ้ายังไม่มี
3. เติมแถวผลงานทีละรายการ (ลำดับ, ชื่อผลงาน, ระดับฐานข้อมูล, บทบาท, คะแนน %)
4. เติมแถวสรุปคะแนนรวมที่ท้ายตาราง

> ⚠️ **หากแก้ไข layout เทมเพลต** (เพิ่ม/ลบคอลัมน์ตาราง, เปลี่ยนคำใน header) ต้องตรวจสอบว่ายังคงคำค้นหา ("ลำดับ"/"ผลงาน"/"ชื่อ" สำหรับตาราง, "เรื่อง"+"ขอเสนอผลงาน" สำหรับย่อหน้าหัวเรื่อง) ไว้ในตำแหน่งเดิม ไม่เช่นนั้นสคริปต์จะหาตำแหน่งแทรกข้อมูลไม่เจอและใช้ fallback ที่อาจไม่ตรงกับ layout จริง

## 6. สิทธิ์การเข้าถึงที่จำเป็น

บัญชี Google ที่ deploy สคริปต์ (หรือบัญชีที่สคริปต์ระบุให้ "รันในนาม" ตามการตั้งค่า Deployment → Execute as) ต้องมีสิทธิ์อย่างน้อยดังนี้:

| ทรัพยากร | สิทธิ์ขั้นต่ำที่ต้องมี |
|---|---|
| Staff Database Sheet | แก้ไขได้ (Editor) — เผื่อกรณีต้องอ่าน/เขียนแบบ dynamic |
| Submissions Database Sheet | แก้ไขได้ (Editor) — เขียนแถวใหม่ทุกครั้งที่มีการยื่น |
| Attachments Root Folder | แก้ไขได้ (Editor) — สร้างโฟลเดอร์ย่อยและไฟล์ใหม่ |
| Memo Template Doc | **อ่านได้อย่างน้อย** (สคริปต์จะ `makeCopy()` ไม่ได้แก้ต้นฉบับโดยตรง) แต่บัญชีต้องมีสิทธิ์สร้างสำเนาได้ |
| Root Folder ของ Drive บัญชีที่รัน (`DriveApp.getRootFolder()`) | ใช้เป็นที่พักไฟล์ `.docx` ชั่วคราวก่อนลบทิ้งใน `generateDocxMemo()` |

การเข้าสู่ระบบของผู้ใช้ปลายทางถูกจำกัดด้วยโดเมนอีเมลใน `doGet()` (`@eng.kmutnb.ac.th` หรือ `@kmutnb.ac.th`) แต่ **ไม่ได้จำกัดว่าใครมีสิทธิ์แก้ไขสเปรดชีต/โฟลเดอร์เหล่านี้โดยตรง** — การจำกัดสิทธิ์แก้ไขไฟล์ต้นทางต้องตั้งค่าแยกที่ระดับ Google Sheets/Drive sharing settings เอง

## 7. งานดูแลระบบตามปกติ

| งาน | ทำที่ไหน | หมายเหตุ |
|---|---|---|
| เพิ่มบุคลากรใหม่ | แท็บ `Staff` | เพิ่มแถวใหม่ กรอกอย่างน้อย `Email`, `Div_ID`, ชื่อ-นามสกุล, ตำแหน่ง |
| เปลี่ยนตำแหน่งทางวิชาการ | แท็บ `Staff` | แก้คอลัมน์ตำแหน่งให้ตรงกับค่าที่ `CRITERIA` รู้จักเป๊ะๆ (ดูรายการใน CLAUDE.md/Code.gs) มิฉะนั้นระบบจะคำนวณเกณฑ์ไม่ได้ |
| เปลี่ยนหัวหน้าภาควิชา | แท็บ `Division` | แก้เฉพาะคอลัมน์อีเมลหัวหน้าภาควิชา |
| เปิดรอบยื่นใหม่ / ปีงบประมาณใหม่ | ไม่ต้องทำอะไรในชีต | โฟลเดอร์และแท็บสร้างอัตโนมัติเมื่อมีการยื่นครั้งแรกของรอบนั้น |
| ตรวจสอบคำขอที่มีปัญหา | แท็บ `Submissions` + `Publications` (คู่กันด้วย `Email`+`Fiscal_Year`+`Round`) | ใช้ `getSubmissionDebugInfo(email)` ในตัวแก้บั๊ก (เรียกจาก Apps Script Editor) เพื่อไล่ดูว่าอีเมลไหนมีกี่แถวจับคู่กันบ้าง |
| จัดเก็บ/archive ปีเก่า | ย้ายโฟลเดอร์ `ปีงบประมาณ {ปีเก่า}` ออกจาก Root Folder ไปเก็บที่อื่น | ไม่กระทบข้อมูลในสเปรดชีต เพราะลิงก์ไฟล์อ้างอิงด้วย Drive file ID ไม่ใช่ path |
| สำรองข้อมูล | File → Make a copy หรือ Download ทั้ง 2 สเปรดชีต เป็นระยะ | ไม่มีระบบ backup อัตโนมัติในตัวสคริปต์ |

## 8. การแก้ไขปัญหาเบื้องต้นสำหรับผู้ดูแลระบบ

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|---|---|---|
| ผู้ใช้ล็อกอินแล้วหน้าโปรไฟล์ว่างเปล่า/error "ไม่พบแท็บ Staff" | แท็บชื่อไม่ตรง `Staff` เป๊ะๆ, หรือถูกลบ/เปลี่ยนชื่อ | เช็คชื่อแท็บในสเปรดชีต Staff Database ให้ตรงตัวพิมพ์ใหญ่-เล็ก |
| ผู้ใช้มีอีเมลถูกต้องแต่ระบบบอกว่า "ไม่พบข้อมูลบุคลากร" | ไม่มีแถวใน `Staff` ที่ Email ตรงกัน (case-insensitive แต่ต้องสะกดตรง) | เพิ่ม/แก้แถวใน Staff ให้ตรงกับอีเมล Google Workspace จริง |
| ยื่นคำขอไม่สำเร็จ "ไม่สามารถเปิด Google Sheet สำหรับบันทึกคำขอได้" | บัญชีที่รันสคริปต์ไม่มีสิทธิ์แก้ไข Submissions Database (เช่นถูกเอาออกจากรายชื่อ Editor) หรือ Sheet ID เปลี่ยน | ตรวจสอบสิทธิ์ Editor ของสเปรดชีตนั้น |
| ไฟล์แนบหายไปจากลิงก์ในสเปรดชีต แต่มีข้อความ "ไฟล์แนบ" ปรากฏ | คอลัมน์ `Attachments` ถูกวางทับด้วยข้อความธรรมดา ทำให้ rich-text runs หาย | ให้ผู้ใช้ยื่นซ้ำ (overwrite) รอบนั้นใหม่ เพื่อให้สคริปต์เขียน rich-text link ใหม่ |
| หัวหน้าภาควิชาที่ปรากฏในบันทึกข้อความผิดคน | อีเมลหัวหน้าภาควิชาในแท็บ `Division` ไม่อัปเดต หรือชื่อภาควิชาสะกดไม่ตรงกับที่ normalize ได้ (`normalizeDept`) | แก้อีเมลใน `Division`, ตรวจว่า `Div_ID` ของ `Staff` ตรงกับ `Div_ID` ของ `Division` |
| การสร้างเอกสาร `.docx` ล้มเหลว "ไม่สามารถแปลงเอกสารเป็น DOCX ได้" | สิทธิ์ OAuth ของสคริปต์หมดอายุ/ไม่ครอบคลุม Drive export, หรือ Template Doc ID ถูกลบ/ย้าย | ตรวจสอบว่า Template Doc ยังอยู่ที่ ID เดิมและบัญชีสคริปต์เข้าถึงได้ ลอง re-authorize สคริปต์ |
| โฟลเดอร์ไฟล์แนบใหม่ไม่ถูกแยกตามปี/รอบ/ชื่อ (ไฟล์ไปกองที่ root) | Root Folder ID ผิด หรือบัญชีสคริปต์ไม่มีสิทธิ์สร้างโฟลเดอร์ย่อยใน Root Folder | ตรวจสอบสิทธิ์ Editor บน Attachments Root Folder |
